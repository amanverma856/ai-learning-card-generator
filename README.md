# AI Learning Card Generator

A full-stack web application that generates structured learning cards for any topic using AI, streamed progressively via WebSockets.

Built for the Ainstein Full Stack Intern Technical Trial.

---

## Features

- Enter any topic and generate 3 learning cards instantly
- Cards stream one by one via WebSocket (not all at once)
- Each card has a Title, Key Concept, and Fun Fact
- Loading skeletons appear while cards generate
- Success Mode: all 3 cards stream successfully
- Failure Mode: Card 3 fails intentionally with an error state
- Retry button re-uses the same WebSocket connection and succeeds
- Responsive UI with dark mode support

## Setup Instructions

### Prerequisites
- Node.js v18 or above
- A Gemini API key (free at https://aistudio.google.com)

### Step 1 - Clone and install

```bash
git clone <your-repo-url>
cd ainstein-task

# Install backend dependencies
cd backend
npm install

# Install frontend dependencies
cd ../frontend
npm install
```

### Step 2 - Add your API key

```bash
cd backend
cp .env.example .env
```

Open `.env` and add your Gemini API key:
```
GEMINI_API_KEY=your_actual_key_here
PORT=8080
```

### Step 3 - Start the backend

```bash
cd backend
npm start
```

You should see: `WebSocket server running on ws://localhost:8080`

### Step 4 - Start the frontend

Open a new terminal:
```bash
cd frontend
npm run dev
```

Open your browser at: `http://localhost:5173`

## How the WebSocket Flow Works

1. The React frontend connects to `ws://localhost:8080` on page load.
2. When the user clicks Generate, the frontend sends:
   ```json
   { "type": "generate", "topic": "Photosynthesis", "mode": "success" }
   ```
3. The backend receives this, calls the Gemini API 3 times (once per card).
4. After each card is generated, the backend sends it immediately:
   ```json
   { "type": "card", "data": { "index": 0, "title": "...", "concept": "...", "funFact": "..." } }
   ```
5. The frontend receives each card and renders it as it arrives.
6. When all cards are sent, the backend sends `{ "type": "done" }`.
7. The frontend shows a completion message.

The connection stays open for the entire session so Retry works without reconnecting.

## How Failure and Retry Works

### Failure Mode
When mode is set to "failure":
- Card 1 generates and streams normally
- Card 2 generates and streams normally
- Card 3 is intentionally skipped — an error message is sent instead:
  ```json
  { "type": "error", "index": 2, "message": "AI generation failed for this card. Please retry." }
  ```
- Cards 1 and 2 remain visible on screen
- Card 3 shows an error state with a Retry button

### Retry
When the user clicks Retry:
- Frontend sends `{ "type": "retry", "topic": "...", "mode": "success" }`
- The same WebSocket connection is used (no reconnect)
- Backend generates only Card 3 and streams it back
- Card 3 renders successfully and the done banner appears

## Assumptions and Design Decisions

- Used Gemini 1.5 Flash as the AI model since it has a free tier and is fast enough for this use case
- Each card is generated by a separate API call with a specific prompt for that card's focus area (Introduction / Core Mechanics / Real World Impact) rather than generating all 3 in one call. This makes streaming feel more natural since each card arrives independently
- Loading skeletons appear progressively (not all 3 at once) to reinforce the streaming feel
- The WebSocket connection is established once on page load and reused for all interactions including Retry
- Dark mode is supported automatically via CSS media queries
